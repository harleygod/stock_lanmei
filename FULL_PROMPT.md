## 一、项目背景与目标

我是一个普通 A 股散户，对数字不敏感，容易做出情绪化决策。

需要一个帮我 **「算清楚再决策」** 的个人投资管理 Web App：

- **核心**：数字可视化 + 决策提示 + 精确手续费计算
- **哲学**：融入查理·芒格式的数学思维（期望值、盈亏比、Kelly 仓位、排列组合情景、反事实检验）
- **部署**：可运行在 VPS 上，Docker 一键部署
- **数据**：浏览器 localStorage 持久化，无需数据库和用户登录

**免责声明（页脚必须展示）：** 本工具仅做数学辅助与决策结构化，不构成投资建议。行情数据来自公开接口，可能有延迟或误差。

## 四、手续费标准（所有计算模块必须统一引用）

用户实际券商：**华宝证券**

| 项目 | 费率 |
|------|------|
| 佣金 | 0.01154%（沪深相同，即 0.0001154） |
| 过户费 | 0.002%（**仅沪市**：主板 + 科创板） |
| 印花税 | 0.1%（**仅卖出**，国家强制，不可修改，即 0.001） |

**买入总费率：**
- 沪市：0.01154% + 0.002% = **0.01354%**
- 深市：0.01154% = **0.01154%**

**卖出总费率：**
- 沪市：0.1% + 0.01154% + 0.002% = **0.11354%**
- 深市：0.1% + 0.01154% = **0.11154%**

**佣金计算：**

```typescript
function calcCommission(amount: number, settings: FeeSettings): number {
  const raw = amount * settings.commissionRate;
  if (settings.minCommission) return Math.max(raw, 5);
  return raw;
}
```

**用户可在设置页修改：**
- 佣金费率（默认 0.0001154）
- 是否启用最低佣金 5 元（开关，**默认关闭**）
- 印花税固定展示，不可修改

## 五、核心计算公式（必须精确实现）

### 5.1 买入总成本

```
买入金额 = 买入价 × 股数
买入手续费 = 佣金 + 过户费(沪市)
买入总成本 = 买入金额 + 买入手续费
```

### 5.2 卖出到手净额

```
卖出金额 = 卖出价 × 股数
卖出手续费 = 印花税 + 佣金 + 过户费(沪市)
到手净额 = 卖出金额 - 卖出手续费
```

### 5.3 浮动盈亏（主页、详情页必须用此公式）

```
浮动盈亏 = 当前价卖出到手净额 - 买入总成本(含买入费)
盈亏比例 = 浮动盈亏 / 买入总成本 × 100%
```

⚠️ **禁止**用 `(现价 - 成本价) × 股数` 忽略手续费，否则与交割单对不上。

### 5.4 回本需涨幅（含卖出费用）

```
卖出费率 = 印花税 + 佣金 + 过户费(沪市)
目标价 = 成本价 × (1 + 卖出费率)
回本需涨幅 = (目标价 - 当前价) / 当前价 × 100%    （当前价 >= 目标价时显示 0 或「已盈利」）
```

⚠️ **禁止**用 `(成本价 - 现价) / 现价`，会低估实际所需涨幅。

### 5.5 亏损后不对称回本（止损模块展示）

```
亏损 r% 后，回本所需涨幅 = r / (1 - r) × 100%

示例：
  亏 10% → 需涨 11.11%
  亏 20% → 需涨 25%
  亏 50% → 需涨 100%
```

### 5.6 期望值（芒格核心）

```
EV = (上涨概率 × 预期涨幅) - (下跌概率 × 预期跌幅)
下跌概率 = 100% - 上涨概率
盈亏比 = 预期涨幅 / 预期跌幅
```

**决策建议：**
- EV > 5% 且 盈亏比 > 2 → ✅ 期望值良好，可以考虑入场
- EV 0%–5% → ⚠️ 期望值一般，建议等更好机会
- EV < 0% → ❌ 期望值为负，赔率不利，不建议操作

### 5.7 Kelly 公式（仓位建议）

```
b = 预期涨幅 / 预期跌幅
f* = (b × p - q) / b
其中 p = 上涨概率, q = 1 - p
输出 f* 和建议「半 Kelly」= f* / 2（更保守）
```

### 5.8 HHI 集中度指数（主页）

```
HHI = Σ(单股市值 / 总市值)²
```

- HHI < 0.15：分散良好
- 0.15–0.25：中等集中
- > 0.25：高度集中，需警告

### 5.9 加权平均成本（多笔买入）

```
对每笔买入分别计算含手续费的 totalCost
加权成本价 = Σ(totalCost) / Σ(股数)
```

### 5.10 目标盈利反推卖出价

用二分法求解：找到卖出价 P 使得 `netSellProceeds(P) - buyTotalCost = 目标盈利`

### 5.11 止损价反推

用二分法求解：找到卖出价 P 使得 `buyTotalCost - netSellProceeds(P) = 最大可承受亏损`

#### Tab 7：机会成本对比（v1.1 新增）

顶部说明：「持有的代价 = 放弃的最佳Alternative。比较持有 A 与换仓 B，哪个期望值更高。」

**输入两组：**

| | 选项 A（当前持有） | 选项 B（换仓目标） |
|--|--|--|
| 当前市值/投入 | 自动从持仓选或手填 | 手填 |
| 上涨概率 | % | % |
| 预期涨幅 | % | % |
| 预期跌幅 | % | % |
| 换仓成本 | — | 买卖手续费合计（自动算） |

**输出：**
- A 的 EV vs B 的 EV（换仓后）
- EV 差值 = B_EV - A_EV
- 决策建议：
  - 差值 > 3% → 「换仓期望值显著更高，可考虑调仓」
  - -3% ~ 3% → 「两者接近，换仓收益不足以覆盖成本和不确定性」
  - 差值 < -3% → 「持有 A 期望值更高，勿因短期波动频繁换仓」

底部金句：「机会成本是看不见的代价 — 每个『持有』都是对另一个选择的放弃。」

---

#### Tab 8：安全边际计算器（v1.1 新增，格雷厄姆/芒格）

顶部说明：「只在价格显著低于保守估值时买入，留出犯错空间。」

**输入：**
- 保守估计每股价值（元）（用户主观估值）
- 安全边际（%，默认 30%）
- 计划买入价（元，可填当前价）
- 股数、交易所

**计算：**
```
最高可接受买入价 = 保守估值 × (1 - 安全边际%)
当前价格折扣率 = (保守估值 - 买入价) / 保守估值 × 100%
是否在安全区内 = 买入价 <= 最高可接受买入价
```

**输出：**
- 最高可接受买入价（大字）
- 当前折扣率（正数绿色=有安全边际，负数红色=溢价买入）
- 是否在安全区内（✅/❌）
- 若当前溢价 X%，需下跌 X% 才进入安全区

底部金句：「以合理价格买伟大公司，好过以伟大价格买合理公司 — 但首先要留出安全边际。」

---

#### Tab 9：手续费拖拽曲线（v1.1 新增）

**输入：** 买入价、股数、交易所

**图表（Recharts 折线图）：**
- X 轴：目标盈利金额（0 ~ 买入成本 × 50%）
- Y 轴：手续费占毛利润比例 (%)
- 曲线：`feeRatio = 总手续费 / (净利润 + 总手续费) × 100%`

**标注点：**
- 手续费占比 = 50% 时对应的目标盈利（「赚 100 块要交 100 块手续费」临界点）
- 当前常用目标盈利点（如 500 元、1000 元）在曲线上的位置

**洞察文字：**
- 「小目标盈利时，手续费侵蚀严重，需更高涨幅才值得操作」
- 「建议单次交易目标盈利 > ¥XXX，手续费占比才低于 20%」

### 模块三：单股详情 + 决策助手（`/position/:id`）

**区域一：基本数据**

- 对总资金的影响比例

**区域三：决策自检清单（5 条，逐条勾选）**

- □ 如果现在手里是现金，我还愿意在这个价格买入吗？
- □ 这只股票的基本面（业绩、行业）有没有变坏？
- □ 我买这只股票的原始逻辑还成立吗？
- □ 我现在持有是因为数据支持，而不是不想承认亏损？
- □ 这只股票的仓位在合理范围内（< 30%）？

**建议逻辑：**
- 5 个全勾 → 绿色：「持有逻辑充分，继续持有」
- 3–4 个 → 黄色：「建议重新评估仓位比例」
- ≤ 2 个 → 红色：「建议认真考虑止损或减仓」

## 九、数据结构（localStorage）

**Storage Key:** `stock-decision-assistant-v1`

```typescript
type Market = 'sh' | 'sz' | 'bj';

type Board =
  | 'sh_main'   // 沪市主板
  | 'sz_main'   // 深市主板

  commissionRate: number;   // 0.0001154

  theme: 'light' | 'dark';

interface Position {
  id: string;               // UUID
  name: string;
  code: string;             // 纯数字，如 "002475"

  note?: string;

  updatedAt: string;
}

interface TradeLog {
  id: string;
  date: string;               // YYYY-MM-DD
  stockName: string;
  stockCode: string;
  action: 'buy' | 'sell' | 'add' | 'reduce';
  price: number;
  quantity: number;
  fee?: number;               // 可选，实际交割单手续费

  low: number;
  volume: number;
  isClosed: boolean;
  source: 'sina' | 'tencent' | 'manual';
  updatedAt: string;
}

interface AppData {
  version: 1;
  positions: Position[];
  logs: TradeLog[];
  settings: AppSettings;
}
```

## 十三、验收标准

1. 添加持仓后，盈亏金额与手动按交割单计算误差 < 0.02 元
2. 回本需涨幅 > 简单 `(成本-现价)/现价`（验证手续费已计入）
3. 新浪接口失败时自动切腾讯，两者均失败时手动模式可用
4. 交易日志不填理由无法提交
5. 导出 JSON → 清空 → 导入，数据完整恢复
6. 移动端 375px 宽度下所有页面可用
7. Docker 构建后单容器可访问全部功能
8. v1.1：机会成本 Tab 换仓 EV 差值计算正确；安全边际判断逻辑正确；手续费曲线在目标盈利 = 0 时趋近 100%
